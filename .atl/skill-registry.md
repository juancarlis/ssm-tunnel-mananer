| --- | --- | --- |
| `ssm-tunnel-manager-conventional-commits` | Preparing a git commit, drafting a commit message, or being asked to commit changes | Use for every future commit message in this project |

## Compact Rules

### ssm-tunnel-manager-conventional-commits

- Use Conventional Commits for every commit in this project.
- Format commit subjects as `<type>: <description>`.
- Pick the type by change intent: `fix`, `feat`, `docs`, `test`, `refactor`, `chore`, `ci`.
- Keep subjects concise, imperative, and lowercase.
- Never add trailing periods.
- Never add `Co-Authored-By` or AI attribution.
